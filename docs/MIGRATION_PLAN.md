# Migration Plan

## 1. Principle

Eski WordPress/WooCommerce — **source**, lekin yangi platforma uchun haqiqat manbasi emas. Import qilingan kontent reviewdan o‘tadi.

## 2. Pipeline

```text
Discovery
→ Raw extraction
→ Immutable raw JSON
→ Media download
→ Normalize/sanitize
→ Mapping
→ Validation report
→ Staging import
→ Human review
→ Publish selection
→ Redirect generation
→ Production import
→ Verification
```

## 3. Access priorities

1. WordPress REST API.
2. WooCommerce Store API/public endpoints.
3. Authenticated WooCommerce REST API.
4. WordPress XML export.
5. Database dump + uploads archive.
6. HTML crawler fallback.

## 4. Extracted objects

- site metadata;
- pages;
- posts;
- categories/tags;
- media;
- products;
- product categories/tags;
- attributes;
- menus where accessible;
- SEO plugin metadata where accessible;
- source URL and timestamps.

## 5. Review state

- `PENDING_REVIEW`
- `APPROVED`
- `NEEDS_UPDATE`
- `ARCHIVED`
- `REJECTED`

## 6. Redirect rules

- source path unique;
- exact product/page mapping preferred;
- discontinued product → best approved alternative or category;
- no mass redirect to homepage;
- redirect loops prevented;
- 404 report after launch.

## 7. Media

- original source URL preserved;
- content hash calculated;
- duplicate binary stored once;
- unsafe/unsupported files rejected;
- alt/caption imported but reviewed;
- variants generated by new media pipeline.

## 8. Idempotency

Har bir legacy object uchun:

```text
sourceSystem + objectType + legacyId
```

unique bo‘ladi. Qayta import update/upsert qiladi, duplicate yaratmaydi.

## 9. Verification

- extracted vs imported counts;
- missing media;
- broken internal links;
- invalid HTML;
- duplicate slugs;
- unknown categories/attributes;
- empty translations;
- redirects coverage;
- products never auto-published;
- random sample visual comparison.
